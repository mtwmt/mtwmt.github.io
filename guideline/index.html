💡 圓角設計原則

  • 6px (sm)：小型元素，細緻柔和
  • 8px (default)：標準互動元素，平衡感最佳
  • 12px (lg)：中大型容器，視覺友善
  • 16px (xl)：大型區塊，強烈視覺層次

📐 Card Manga 風格設計規範 (Comic Book Style)

視覺樣式：
  • 圓角：8px rounded-md (適中硬朗)
  • 邊框：2px border-gray-900 (粗黑線條)
  • 背景：bg-white (純白高對比)
  • 陰影：4px 4px 0 rgba(0,0,0,0.15) (硬陰影偏移)
  • 標題：font-bold (粗體強調)
互動效果：
  • 上移距離：-translate-y-1 (明顯動態)
  • 陰影增強：6px 6px 0 (更大偏移)
  • 過渡時間：200ms (快速反應)
  • 硬陰影特色：無模糊效果，純偏移

💡 Comic Book 風格精髓：粗黑邊框 + 硬陰影偏移 + 高對比 + 明確層次

🌗 暗色模式特別處理

暗色模式下，陰影從黑色反轉為白色：

  • 亮色模式：4px 4px 0 rgba(0,0,0,0.15)
  • 暗色模式：4px 4px 0 rgba(255,255,255,0.1)

白色陰影在深色背景上才能顯示層次感

🏷️ Tag vs Badge 標籤系統

兩種不同用途的標籤組件

💡 Tag 與 Badge 的差異

Tag（標籤）
  • 用途：內容分類、導航
  • 外觀：方形 rounded-md
  • 互動：可點擊，有 # 前綴
  • 範例：部落格文章標籤
Badge（徽章）
  • 用途：狀態指示、標記
  • 外觀：圓形 rounded-full
  • 互動：通常不可點擊
  • 範例：技能標記、狀態顯示

Tags（標籤）組件

用於文章分類、導航的標籤

#JavaScript #Astro #TailwindCSS #TypeScript 
<Tags tags=JavaScriptAstroTailwindCSS />

Badge（徽章）組件

用於狀態指示、技能標記的徽章

顏色變體

Primary Accent Success Warning Error Info Neutral 
<Badge color="primary|accent|success|...">...</Badge>

外框樣式

Primary Accent Success 
<Badge outline=true>...</Badge>

尺寸

Small Medium Large 
<Badge size="sm|md|lg">...</Badge>
