淺談AMP

AMP 是 GOOGLE 力推的一項技術。 重點就是「快」 目前大部份使用於行動裝置上

優勢

速度快

  • 當 GOOGLE 的爬蟲爬到你的 AMP 頁面時 他會把你的網頁自動 CACHE 到 GOOGLE 的伺服器存放一份，因此再次載入時會先從 GOOGLE 的伺服器裡載入，更有效率的快速載入頁面
  • 同時限制 CSS 圖片 多媒體 等檔案大小，也支援異步加載(不過都要做 AMP 了 JS 這部份當然能省就省了)

SEO 效果好

  • 更精準的投放到對產品有興趣的客層

提高 GOOGLE 搜尋排名

缺點

使用自訂的 TAG 所以開發者必需依照 AMP 的規範撰寫 (廢話 XD

有些人會有疑問 快取都做在 GOOGLE 的 SERVER 裡了 流量會不會也算 GOOGLE 的，答案是會的，但相反的我們使用者想知道的只是有多少人來過，所以在開發上可以在頁面裡埋事件去計算
